## 📦 Platform Support

### Installation Commands

#### Python (pip)
```bash
pip install kore-fileformat==1.1.5
```
- Requires Python 3.8+
- Pre-built wheels for: Windows, macOS (Intel/ARM), Linux
- Pure Rust implementation via PyO3

#### JavaScript/Node.js (npm)
```bash
npm install kore-fileformat@1.1.5
```
- Requires Node.js 12+
- Native binding (NAPI)
- Works in Node.js and Electron environments

#### Java (Maven)
```xml
<dependency>
  <groupId>com.korefileformat</groupId>
  <artifactId>kore-fileformat</artifactId>
  <version>1.1.5</version>
</dependency>
```
- Requires Java 8+
- JAR file deployed to Maven Central (24-48 hour sync window)
- Works with Gradle, Maven, Sbt, etc.

#### Rust (Cargo)
```toml
[dependencies]
kore-fileformat = "1.1.5"
```
- Core library
- Works with all Rust versions (1.56+)
- No external dependencies

#### Docker
```bash
docker pull ghcr.io/arunkatherashala/kore:1.1.5
```
- Multi-language reference image
- Includes: Rust, Python, JavaScript, Java, Go
- Development and testing environment

## 🚀 Getting Started

### Python Example
```python
from kore_fileformat import KoreWriter

# Create a Kore file
writer = KoreWriter()
writer.add_column("values", [1, 2, 3, 4, 5])
writer.write("output.kore")

# Read it back
from kore_fileformat import KoreReader
reader = KoreReader("output.kore")
data = reader.read()
```

### JavaScript Example
```javascript
const { KoreWriter } = require('kore-fileformat');

const writer = new KoreWriter();
writer.addColumn('values', [1, 2, 3, 4, 5]);
writer.write('output.kore');

const { KoreReader } = require('kore-fileformat');
const reader = new KoreReader('output.kore');
const data = reader.read();
```

### Java Example
```java
import com.korefileformat.KoreWriter;

KoreWriter writer = new KoreWriter();
writer.addColumn("values", new int[]{1, 2, 3, 4, 5});
writer.write("output.kore");

KoreReader reader = new KoreReader("output.kore");
byte[][] data = reader.read();
```

## 📋 Known Limitations

### v1.1.5 Scope
- Single-threaded compression (multi-threading planned for v1.2.0)
- Fixed schema (no schema evolution in v1.1.x)
- No built-in encryption (use OS-level encryption)
- No compression level tuning (automatic selection only)

### Planned for Future Releases
- v1.1.6: Multi-threaded compression
- v1.1.7: Schema evolution support
- v1.1.8: Custom compression profiles
- v1.2.0: Advanced features (partitioning, filtering, querying)
